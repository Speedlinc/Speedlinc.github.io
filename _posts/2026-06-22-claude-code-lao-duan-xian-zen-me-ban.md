## Claude 为什么总是断线

Claude 的网页对话和 Claude Code 这类工具，都需要和海外服务器保持一条持续、稳定的连接。和普通网页不同，AI 工具往往要长时间保持会话：读一份长文档、跑一段代码、连续追问，背后是一条一直开着的通道。这条通道只要中途丢包或被中转打断，就会表现为掉线、报错、任务中断。常见原因有三类：

- **长连接被中断**：公共线路不稳，长会话中途断开，Claude Code 跑长任务时尤其明显。
- **延迟忽高忽低**：线路抖动让响应一会儿快一会儿慢，写代码补全经常卡顿。
- **出口 IP 连接受限**：共享出口被太多人使用，触发临时限制甚至访问不可用。

### 网页端和命令行端的区别

很多人只在网页端遇到问题，其实命令行工具更敏感。网页对话断了刷新一下还能接着聊，但 Claude Code、Cursor 这类工具在跑 agent 任务、连续调用接口时，一旦连接抖动，整个任务可能就失败重来。所以对开发者来说，链路稳定比单纯的速度更重要。
